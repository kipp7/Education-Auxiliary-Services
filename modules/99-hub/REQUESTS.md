# 跨模块请求队列

当某模块需要改动别的模块目录内的文件时，请在这里新增一条请求，由总控窗口统一处理。

模板：

```markdown
## 请求标题
- 发起模块：`modules/xx-...`
- 目标模块：`modules/yy-...`
- 原因：
- 期望改动：
- 影响评估：
```

## 首页重做（多题库导入 + 学生刷题）任务拆解
- 发起模块：modules/03-importer（产品建议）
- 目标模块：modules/01-miniapp
- 原因：项目定位为“多题库导入 + 学生自主练题”，首页需以“选题库/继续练习/快速开刷”为核心而非内容运营
- 期望改动（建议写入 01 的 TASKS.md）：
  - 首页第一屏：当前题库选择器（最近使用/快速切换）+ 主按钮“继续练习/开始练习”
  - 快捷入口：按章节/按题型/错题本/收藏（可选：模拟测验）
  - 导入入口：题库管理页（列表：我的题库；动作：导入/重命名/删除/统计）在首页可达（但不抢主CTA）
  - 学习反馈：今日刷题数/连续天数/正确率（简版即可）
  - 空状态：无题库时引导“导入题库”+ 示例题库/教程
- 验收方式：
  - 新用户无题库：进入首页能在 1 次点击内到达导入流程入口
  - 有题库有历史：进入首页 1 次点击内进入“继续练习”并显示题库名/进度
  - 切换题库后，章节/错题/继续练习均作用于当前题库（顶部显式展示当前题库）
