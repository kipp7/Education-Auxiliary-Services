# 04-admin-console AUTOPILOT

> 目的：把本模块按里程碑拆成可连续交付的小步；每完成一个里程碑：
> 1) commit + push
> 2) 追加验收路径到 `modules/04-admin-console/CONVERSATION_LOG.txt`
>
> 约束：本模块实现以 99-hub 的 UI/权限/接口契约为准；在契约未冻结前允许占位（本地示例数据 / dev 登录）。

## M1：管理台 Web 骨架 + 登录与权限守卫

- 产出：
  - `modules/04-admin-console/web/`：Vite + React + TS 最小可运行
  - `/login`：开发占位登录
  - `/`：RequireAuth 保护的首页
- 验收路径：
  - `cd modules/04-admin-console/web`
  - `npm install`
  - `npm run dev`
  - 打开 `http://localhost:5174`，点击“开发登录”进入首页；退出后回到登录页

## M2：题库管理（最小骨架）

- 产出：`/question-bank` 页面（科目/题库包/章节/题目列表，占位 CRUD）
- 验收路径：
  - 登录后进入“题库管理”
  - 可新增/重命名/删除：科目、题库包、章节、题目

## M3：导入管理（最小骨架）

- 产出：`/imports` 与 `/imports/:taskId`（上传占位、任务列表、详情、错误报告下载占位）
- 验收路径：
  - 登录后进入“导入管理”
  - 点击“上传（占位）”创建任务，进入详情页，可下载错误 CSV（占位）

## M4：资讯/公告管理（最小骨架）

- 产出：`/cms`（公告新增/编辑/删除、草稿/发布切换、搜索，占位数据）
- 验收路径：
  - 登录后进入“公告管理”
  - 可新增公告、编辑、发布/下线、搜索、删除

## M5：SVIP 码管理（最小骨架）

> 覆盖 `TASKS.md` 中“用户与权益管理（VIP/激活码）”的最小可用部分：激活码生成/导出/状态流转（占位实现）。

- 产出：`/svip-codes`（批量生成、列表、导出 CSV、作废/注销，占位数据）
- 验收路径：
  - 登录后进入“SVIP 码管理”
  - 可批量生成若干码；可导出 CSV；可将单个码标记为“已注销”；可重置示例数据

## M6：数据看板（最小骨架）

> 覆盖 `TASKS.md` 中“数据看板（活跃、正确率、完成率）”的占位实现：先提供可运行的 Dashboard 页面与基础指标卡片。

- 产出：`/dashboard`（聚合本地占位数据的概览指标：题库数量、导入任务、公告、SVIP 码等）
- 验收路径：
  - 登录后进入“数据看板”
  - 可看到指标卡片；刷新页面仍可读取 localStorage 的最新数据

## M7：轮播图管理（最小骨架）

> 覆盖《需求规格说明书》的“轮播图管理”：上传/排序/跳转链接/轮播开关/间隔（占位实现）。

- 产出：`/banners`（新增/编辑/删除、启用开关、排序、跳转链接、全局间隔设置，占位数据）
- 验收路径：
  - 登录后进入“轮播图管理”
  - 可新增轮播项（图片 URL/跳转链接）
  - 可上移/下移排序、启用/停用、删除
  - 可设置轮播间隔（秒）并刷新页面验证持久化

## M8：课程视频管理（最小骨架）

> 覆盖《需求规格说明书》的“课程视频管理”：学段/年级/科目/单元关联，封面/标题/简介/时长（占位实现）。

- 产出：`/courses`（课程条目 CRUD、基础筛选，占位数据）
- 验收路径：
  - 登录后进入“课程视频管理”
  - 可新增课程（学段/年级/科目/单元、封面 URL、标题、简介、时长）
  - 可编辑/删除、按关键字过滤、刷新页面验证持久化

## M9：试题管理（最小骨架）

> 覆盖《需求规格说明书》的“试题管理”：按学段/科目/单元/题型归档，支持批量导入/导出（占位实现）。

- 产出：`/questions`（试题条目 CRUD、筛选、导出 CSV，占位数据）
- 验收路径：
  - 登录后进入“试题管理”
  - 可新增试题（学段/科目/单元/题型、题干）
  - 可编辑/删除、按关键字筛选、导出 CSV、刷新页面验证持久化

## M10：与契约对齐的“假数据联动”（可选，适合挂机加量）

> 目标：管理台能导出一份“配置 JSON”，小程序侧可读这个 JSON 当作 Mock（先不接真实后端）。

- 轮播图导出（banner 列表）
- 题库树导出（学段/年级/科目/单元）
- 视频列表导出（按单元关联）

## M11：运营与商业配置导出（可选，适合挂机加量）

> 目标：为小程序首页/资讯提供可替换的运营配置（先 JSON 导出，后续接后端）。

- 资讯列表导出（news 列表/详情占位）
- 推荐位导出（recommendations：题库/课程/套餐）
- 套餐导出（plans：名称/价格/有效期/权益范围，占位）
